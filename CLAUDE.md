# Проект: бот Паола (iozefson-bot)

Контекст, правила и схема работы для Claude Code. Читать при каждом старте сессии.

---

## Стартовый чеклист

При каждой новой сессии:
1. `git pull origin master` — синхронизировать локальный репо с GitHub
2. Прочитать техдокументацию через Notion MCP (ссылка ниже)
3. Прочитать инструкции для Claude Code (ссылка ниже)
4. Уточнить у Юлии — что делаем сегодня

Notion MCP доступен в сессии — читать страницы напрямую через него.

---

## Репозиторий

- **Основная ветка:** `master` — деплоится на Railway
- **Рабочие ветки:** `claude/название-задачи` → мерж в `master`
- **Основной файл:** `bot.py` (~2514 строк, весь код бота)
- Остальные файлы: `auth_google.py`, `requirements.txt`, `Procfile`
- **`paola-app/`** — отдельное веб-приложение «Паола App» (рутина + аналитика): FastAPI, агентное ядро на tool use, свои Procfile и requirements. Деплоится вторым сервисом Railway (Root Directory `paola-app`), кода бота не касается. Подробности — `paola-app/README.md` и отдельная ветка документации в Notion (см. «Ключевые страницы Notion» ниже) — своя техдокументация, лог итераций и backlog, отвязанные от бота.

## Правила Claude Code

1. **Полная информация** — перечислять ВСЕ затронутые части изменений явно. Не замалчивать смежные эффекты (например, если расписание меняется для нескольких дней — называть каждый).
2. **Подтверждение перед пушем** — после внесения изменений показать краткое саммари (что изменено, какие файлы, что затронуто) и ждать явного "ок" от Юлии перед `git push`.
3. **Деплой — не только push в рабочую ветку.** Railway деплоит только из `master`. Задача считается завершённой, только когда изменения смержены в `master` и запушены туда — это часть того же шага, что и п.2, не отдельное напоминание. После "ок" от Юлии: push в рабочую ветку → merge в `master` → push `master`. Не оставлять изменения висеть только в `claude/название-задачи`.
4. **После сессии — обязательный финальный чеклист** (выполнять без напоминания):
   - [ ] Обновить **лог итераций** в Notion: добавить строку в таблицу сессий, пересчитать аккумулятив, добавить детализацию, обновить типы работ
   - [ ] Обновить **техдокументацию бота** в Notion: структура сводок, количество строк, переменные, платформа — всё что изменилось
   - [ ] Обновить **строку бота** в CLAUDE.md (количество строк `bot.py`) если изменилась
   - [ ] Написать **саммари для Юлии** — коротко в чат: что сделали, что изменилось, что проверить/протестировать

   Если сессия трогала `paola-app/` — лог итераций и техдокументация обновляются в **отдельной ветке Паола App** (см. ниже), не в логе/техдокументации бота. Backlog Паола App — отмечать выполненные пункты по мере продвижения.
