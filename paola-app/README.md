# Паола App — персональный ассистент (рутина + аналитика)

Новое веб-приложение, закрывающее оба слоя ТЗ:

- **Рутина**: сводка по календарю и задачам, привычки со стриками, напоминания
  (пуш в Telegram), фокус дня.
- **Аналитика**: отдельные чаты по функциям — 🤖 Ассистент, 🇮🇹 Переводчик,
  🔎 Ресерч (веб-поиск с источниками), 🏛 Совет директоров (CFO + CMO + инвестор +
  скептичный клиент-PMI), 🧩 Бизнес-модель. Чаты группируются в **проекты**
  (папки по клиентам): контекст проекта передаётся агенту в каждом чате.

Память — те же базы Notion, что у Second Brain: приложение и бот Паола видят
одни и те же задачи. Код бота (`../bot.py`) не затронут.

## Архитектура

```
static/ (веб-интерфейс)  ⇄  FastAPI (main.py)
                              ├─ agent.py — агентный цикл Claude, 5 режимов чатов
                              ├─ storage.py — SQLite: проекты, чаты, история сообщений
                              ├─ tools/  — задачи · календарь · привычки · напоминания
                              │            переводы · совет директоров
                              ├─ services/ — Notion (async) · Google Calendar · Telegram
                              ├─ digests.py — утренняя/вечерняя сводки
                              └─ scheduler.py — напоминания (каждую минуту),
                                                дайджесты 08:00/21:00 (опционально)
```

**Истории чатов** хранятся в SQLite. Чтобы они переживали редеплой, на Railway
подключи Volume (Service → Settings → Volumes → Mount Path `/data`) и добавь
переменную `DATA_DIR=/data`. Без volume чаты живут до следующего деплоя.

Модели: `claude-sonnet-5` (агент, аналитика), `claude-haiku-4-5` (переводы).

## Подготовка Notion (один раз)

Создать три базы и расшарить их интеграции (той же, что у бота):

**Привычки** (`NOTION_HABITS_DB_ID`)
| Свойство | Тип |
|---|---|
| Привычка | Title |
| Активна | Checkbox |
| Цель | Text (необязательно) |

**Журнал привычек** (`NOTION_HABIT_LOG_DB_ID`)
| Свойство | Тип |
|---|---|
| Запись | Title |
| Привычка | Text |
| Дата | Date |
| Выполнено | Checkbox |

**Напоминания** (`NOTION_REMINDERS_DB_ID`)
| Свойство | Тип |
|---|---|
| Напоминание | Title |
| Когда | Date (с временем) |
| Отправлено | Checkbox |

База «Задачи» — существующая, ID тот же, что у бота.

## Локальный запуск

```bash
cd paola-app
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # заполнить значения
uvicorn main:app --reload
# открыть http://127.0.0.1:8000
```

## Деплой на Railway (отдельный сервис, Паолу не трогает)

1. Railway → New Service → тот же GitHub-репозиторий.
2. Settings → **Root Directory: `paola-app`** — сервис видит только эту папку.
3. Variables → скопировать по списку из `.env.example`.
4. Start command подтянется из `paola-app/Procfile`.
5. Networking → Generate Domain — это адрес приложения.

Бот Паола продолжает работать как отдельный сервис из корня репо — они не
мешают друг другу. Когда приложение возьмёт на себя утро/вечер, включить
`DIGESTS_TO_TELEGRAM=1` и отключить соответствующие job'ы у бота.

## Перенос в отдельный репозиторий (когда понадобится)
